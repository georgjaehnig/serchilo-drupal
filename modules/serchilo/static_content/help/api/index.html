[site:name] also provides an API. When calling a shortcut, instead of getting redirected to the shortcut's target URL, you also can let [site:name] only output the URL and some detailed information. This allows you to do you own things with the URL (for instance, piping it to wget on your console or programming your own [site:name] client.)

And instead of passing the search arguments to [site:name], you can also pass only they keyword with the argument count and receive the URL template. This allows you to program your own query parser and to use [site:name] with maximum privacy.

Overview

Examples Receiving URL only as plain text Receiving URL and details in JSON format
Calling with query [site:url]url/n/de.deu?query=g%20berlin [site:url]api/n/de.deu?query=g%20berlin
Calling with keyword and argument_count [site:url]url/n/de.deu?keyword=g&argument_count=1 [site:url]api/n/de.deu?keyword=g&argument_count=1

Calling with keyword and argument_count, you don't need to give your search data to [site:name]. You could instead implement your own query parser and replace the placeholders yourself after receiving the URL from [site:name]. Read more about URL placeholders.
